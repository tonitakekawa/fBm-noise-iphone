# 目論見：fBmノイズの追加

## 概要

既存のホワイトノイズ再生機能に加えて、**fBm（Fractional Brownian Motion）ノイズ**を生成・再生できるようにする。UI には既存のホワイトノイズ再生ボタンと並んで「fBm」ボタンを追加する。

## 背景：fBm とは

fBm は **fractal Brownian motion**（非整数ブラウン運動）。複数のオクターブのノイズを、周波数を倍々に上げ、振幅を徐々に下げながら重ね合わせて生成する。

- スペクトル的には 1/f 系（ピンク〜ブラウンの間）に位置する
- 単に IIR でスペクトル整形したピンクノイズと違い、**自己相似性（時間軸上のフラクタル性）** を持つ
- 自然界の 1/f ゆらぎ（心拍、ろうそくの炎、川の流れ）と同種の構造
- 長時間聴くと、数秒のうねりと数十秒のうねりが入れ子になった「自然な揺らぎ」が知覚できる

## 実装方針

### 1. ノイズ生成器の追加

`WhiteNoisePlayer.swift` と並ぶ形で `FBmNoisePlayer.swift` を新規作成する。
（既存の `WhiteNoisePlayer` の構造を踏襲し、`AVAudioEngine` + `AVAudioSourceNode` ベース）

### 2. fBm 生成アルゴリズム

リアルタイム生成に適した **Voss-McCartney アルゴリズム** を採用する。
spectral synthesis（IFFTベース）は理論的には正攻法だが、ストリーミング再生だと窓の繋ぎ目処理が面倒なので、リアルタイム用途では Voss-McCartney がほぼ標準。

#### Voss-McCartney の原理

- N 個の独立した乱数生成器（オクターブ）を用意する
- 各オクターブ k は `2^k` サンプルごとに更新される
- 各時刻のサンプル値は、全オクターブの現在値の合計
- これにより、低周波成分は緩やかに、高周波成分は速く変化する重ね合わせが得られる
- 結果として 1/f に近いスペクトルかつ、フラクタル的自己相似性を持つ信号になる

#### パラメータ

- `octaves`: オクターブ数（推奨：8〜10）
  - 多いほど低周波の揺らぎが深くなる
  - サンプリングレート 44100Hz で 10 オクターブなら最低更新周期 ≈ 23ms（≈ 43Hz相当の揺らぎ）まで含む
- `gain`: 各オクターブの減衰率（推奨：0.5）
  - 1.0 に近いほど低周波が強くピンク〜ブラウン寄りに
  - 0.5 だとバランスの取れた fBm
- 出力は最後に正規化して `[-1.0, 1.0]` に収める（クリップ防止のため 0.5 倍程度の安全係数を掛ける）

### 3. 実装の擬似コード

```swift
final class FBmGenerator {
    private let octaves: Int
    private let gain: Float
    private var values: [Float]      // 各オクターブの現在値
    private var counter: UInt64 = 0
    private var normalizationFactor: Float

    init(octaves: Int = 8, gain: Float = 0.5) {
        self.octaves = octaves
        self.gain = gain
        self.values = (0..<octaves).map { _ in Float.random(in: -1...1) }
        // 全オクターブが最大値のときの理論値で正規化
        var sum: Float = 0
        var amp: Float = 1
        for _ in 0..<octaves { sum += amp; amp *= gain }
        self.normalizationFactor = 1.0 / sum
    }

    func nextSample() -> Float {
        counter &+= 1
        // どのオクターブを更新するかを決定
        // counter の最下位ビットから連続する 0 の数 = 更新するオクターブのインデックス
        var k = 0
        var c = counter
        while (c & 1) == 0 && k < octaves - 1 {
            c >>= 1
            k += 1
        }
        values[k] = Float.random(in: -1...1)

        // 重み付き合計
        var sum: Float = 0
        var amp: Float = 1
        for v in values {
            sum += v * amp
            amp *= gain
        }
        return sum * normalizationFactor * 0.5  // 安全係数
    }
}
```

`AVAudioSourceNode` のレンダーブロック内で `nextSample()` を呼ぶだけで使える。
ステレオなら左右で別インスタンスを持つと自然な広がりになる（同じインスタンスの値を両チャンネルに流すと中央に定位して窮屈になる）。

### 4. UI の変更

`ContentView.swift` に fBm 用のボタンを追加。既存のホワイトノイズボタンと並べる。

- 排他制御：fBm 再生中はホワイトノイズを停止し、その逆も同様
- 状態管理は `@StateObject` で `FBmNoisePlayer` を保持
- 既存のホワイトノイズの再生ロジックを参考にして同じパターンで書く

## プラットフォーム条件分岐

既存コードと同じく、`#if os(iOS)` で `AVAudioSession` 関連を囲む。
macOS / Mac Catalyst では `AVAudioEngine` のみ使用、セッション設定はスキップ。
（既存の `WhiteNoisePlayer` のパターンを踏襲）

## 動作確認のポイント

1. **聴感**：ホワイトノイズより低音寄りで、波打つような揺らぎが感じられるか
2. **クリップしないか**：レベルメーターまたは出力波形を確認
3. **長時間再生で破綻しないか**：`counter` のオーバーフロー対策として `&+=` を使用済み（u64 なら実用上問題なし）
4. **左右独立性**：ステレオの広がりが自然か

## 任意の拡張（今回は不要、将来用メモ）

- `gain` をスライダーで可変にすると、ピンク〜ブラウン〜fBm を連続的に切り替えられる
- フェードイン/アウト：就寝用途での音量自動低減
- 視覚化：生成中のサンプル値を波形表示すれば、フラクタル性の確認に使える
